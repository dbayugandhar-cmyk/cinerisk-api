# CINEOS Revenue Model

## Seven Revenue Streams

| Stream | Model | Price | Margin |
|--------|-------|-------|--------|
| SaaS Dashboard | Monthly SaaS | Rs 50K/month | 90% |
| Enterprise Sub | Annual contract | Rs 10L-2Cr/year | 85% |
| API Access | Usage-based | Rs 5K-20K/month | 95% |
| Investigation Reports | One-time | Rs 50K-10L | 80% |
| Intelligence Feeds | Data licence | Rs 5L-25L/year | 92% |
| Real-time Alerts | Add-on/usage | Rs 2K+/month | 95% |
| Takedown Services | Per action | Rs 5K-25K | 70% |

## Revenue Scenarios

- 2 enterprise subscriptions: Rs 50L
- 5 investigation reports: Rs 10L

**Total: Rs 71L ARR**

### Moderate (Year 2)
- 5 enterprise + 1 govt contract: Rs 2Cr

- 10 API customers: Rs 30L
- Intelligence feed × 2: Rs 30L
- Takedown retainer × 3: Rs 18L
**Total: Rs 3.18Cr ARR**

### Strong (Year 3)
- 15 enterprise + 3 govt contracts: Rs 8Cr
- 50 investigation reports: Rs 1Cr
- Data partnerships × 3 (ZeroFox etc): Rs 2Cr
- API × 25 customers: Rs 75L
- Takedown retainer × 10: Rs 60L
**Total: Rs 12.35Cr ARR**

## Why This Model Works

1. Data compounds — value grows without proportional cost
2. High margins — intelligence is 85-95% margin
3. Multiple streams — no single point of failure
4. Government + enterprise — low churn, long contracts
5. Network effects — more clients = better intelligence
6. Moat — accumulated data cannot be replicated
